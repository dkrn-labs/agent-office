# LinkedIn Launch Draft

## Main version

I’ve been building a hobby project called **Agent Office**: a local-first workspace for running multiple AI coding agents across real repos without losing track of what each one is doing.

The idea is simple: instead of treating agents like invisible background tabs, give them a visible office. You pick a persona, choose the project, choose the provider and model, inspect the exact prompt before launch, and then watch live status, session history, and portfolio-level output across your local projects.

It currently supports **Claude Code, Codex, and Gemini CLI**, with provider-aware telemetry and a pixel-art office UI that makes the operator side of agent workflows much easier to reason about.

It’s still a hobby project, but the value prop is already clear for me: better launch discipline, better visibility, and less “what is this agent actually doing right now?”
