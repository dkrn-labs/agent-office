# Asset Provenance

This directory contains a mix of:
- bundled third-party visual assets used by the office demo
- generated metadata files created inside this repository

This file exists to make the asset situation explicit for public GitHub distribution.

## Current inventory

### Bundled visual assets

These directories contain shipped pixel-art PNGs used by the office UI:

- `characters/`
- `floors/`
- `walls/`
- `furniture/`

Current provenance status:
- these assets are treated as **third-party bundled art**
- the original planning docs reference LimeZu office / interior art packs as the source family
- redistribution terms for the exact bundled files have **not yet been fully normalized asset-by-asset inside this repo**

That means:
- the assets are fine for hobby/demo development in the current state
- they should **not** be assumed to be covered by the repo's MIT license
- before commercial use or broad redistribution, they should be audited and likely replaced with clearly redistributable assets

## Generated files

These files are generated by repo-owned scripts and are not third-party art:

- `default-layout.json`
  - generated by `scripts/build-default-layout.js`
- `furniture-manifest.json`
  - generated by `scripts/build-furniture-manifest.js`

## Policy for this repo

Treat the code and the bundled art separately:

- root `LICENSE` applies to repo-owned code
- third-party art assets may have different terms
- this repo documents bundled art provenance honestly, but does not yet claim full redistribution certainty for every shipped PNG

## Next hardening steps

To fully harden provenance, future work should:
- map each asset set to an explicit upstream source
- add license text or direct links where required
- replace any asset that is not clearly safe for the intended public distribution model
